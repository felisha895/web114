  1. Why is the JavaScript textContent property safer than innerHTML?

    2. Since textContent only changes the text within an element and innerHTML changes the HTML content within an element, The textContent is safer because in innerHTML, you run the risk of messing up your entire HTML structure then just changing text.

  2. Explain the relationship between an HTML attribute and a corresponding DOM property (explain the connection to JavaScript). Use a specific example in your answer.

    3. You can use HTML attributes like class and id to add it to JavaScript. For example, if we want to target an h1 with class header, we would use the document.getElementsByClassName("header") method to access the h1.
